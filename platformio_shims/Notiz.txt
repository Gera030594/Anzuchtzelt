Bewertung

Codex hat nicht einfach SPI entfernt, sondern einen lokalen PlatformIO-Shim angelegt:

file://platformio_shims/SPI

Damit wird PlatformIO zufriedengestellt, weil eine Library namens SPI vorhanden ist. Die echte SPI-Funktion kommt trotzdem weiterhin aus dem ESP32-Arduino-Framework.

Das ist in diesem Fall sinnvoll, weil:

NeoPixelBus intern SPI als Abhängigkeit deklariert.
ESP32-Arduino bringt SPI.h bereits mit.
PlatformIO versucht sonst fälschlich, SPI extern zu installieren.
Der Shim enthält keine echte Implementierung und überschreibt damit nichts Wichtiges.

was ist so ein shirm
Nachgedacht für eine Sekunde

Ein Shim ist eine kleine Zwischenschicht, die einem Programm etwas „vorspielt“, damit es sauber weiterläuft.

In deinem Fall:

file://platformio_shims/SPI

bedeutet:

PlatformIO sucht eine Library mit dem Namen:

SPI

Weil NeoPixelBus diese Abhängigkeit anmeldet.

Aber beim ESP32 ist SPI schon im Arduino-Framework enthalten. Es muss nicht extra installiert werden.

Was der Shim macht

Der Shim sagt PlatformIO:

Ja, eine Library namens SPI existiert.

Mehr nicht.

Die Datei:

platformio_shims/SPI/library.json

enthält nur Metadaten:

{
  "name": "SPI",
  "version": "0.0.0",
  "description": "PlatformIO shim for the framework-provided ESP32 Arduino SPI library.",
  "frameworks": "arduino",
  "platforms": "espressif32"
}

Das ist keine echte SPI-Library.

Wichtig

Der Shim ersetzt nicht die echte SPI-Funktion.

Die echte Datei:

#include <SPI.h>

kommt weiterhin aus dem ESP32-Arduino-Framework.

Der Shim ist nur ein Platzhalter für PlatformIOs Abhängigkeitsprüfung.

Bildlich

Ohne Shim:

NeoPixelBus sagt: Ich brauche SPI.
PlatformIO sagt: Ich suche SPI im Internet.
PlatformIO scheitert.
Build bricht ab.

Mit Shim:

NeoPixelBus sagt: Ich brauche SPI.
PlatformIO sieht: SPI ist lokal vorhanden.
PlatformIO versucht keinen falschen Download.
ESP32-Arduino liefert die echte SPI.h.
Build läuft.
Warum nicht einfach löschen?

Wenn NeoPixelBus intern SPI als Abhängigkeit deklariert, kann PlatformIO trotzdem versuchen, SPI zu installieren, auch wenn du es nicht direkt in platformio.ini eingetragen hast.

Der Shim fängt genau diesen falschen Installationsversuch ab.
